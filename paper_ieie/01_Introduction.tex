\section{서론}\label{sec:intro}

심층 신경망 추론을 클라우드가 아닌 초소형 마이크로컨트롤러(MCU) 위에서 직접 수행하는 TinyML은 정적 메모리 관리 기반의 경량 런타임과 정수 전용 커널 라이브러리를 중심으로 빠르게 성숙해 왔다\cite{david2021tflm,lai2018cmsisnn,lin2020mcunet}. 한편 스파이킹 신경망(Spiking Neural Network, SNN)은 뉴런 간에 이진 스파이크로 정보를 교환하는 신경망 모델로서\cite{maass1997networks}, 이벤트 기반 연산이 갖는 저전력 잠재력 때문에 엣지 추론의 후보로 주목받아 왔다\cite{roy2019towards}. 그러나 SNN의 배포 연구는 Loihi, TrueNorth와 같은 전용 뉴로모픽 하드웨어에 집중되어 왔고\cite{davies2018loihi,merolla2014truenorth}, 이미 대규모로 보급된 범용 MCU 위에서 SNN을 소프트웨어로 실행하고 그 성능 특성을 실측한 연구는 저자가 아는 한 드물다.

본 논문은 STM32F746(Cortex-M7 200MHz, SRAM 320KB, Flash 1MB) 보드에서 rate coding 기반 SNN의 추론 지연시간과 메모리 사용을 다룬다. 논의에 앞서 본 연구의 범위를 명확히 한다. 본 구현은 스파이크 희소성을 활용하지 않는 dense 연산이며, 에너지 실측은 수행하지 않았다. 즉 본 논문은 SNN의 저전력 잠재력 자체를 검증하는 것이 아니라, 주어진 SNN을 범용 MCU에서 정확하고 빠르게 실행하는 문제에 집중하며, 전력 실측과 희소성 활용(zero-skipping)은 향후 과제로 남긴다.

SNN 추론의 비용 구조는 인공신경망(ANN)과 다르다. rate coding에서 이미지 1장은 $T$개의 이진 스파이크 프레임으로 인코딩되고 네트워크 전체가 $T$회 반복 실행되므로(본 논문 $T{=}30$), 시간축 반복이 지연시간을 지배한다. 여기에 표준 TinyML 스택을 그대로 이식하면 오히려 느려질 수 있음을 본 논문은 실측으로 보인다. LeNet-5급 SNN을 TFLite Micro(TFLM)+CMSIS-NN의 표준 int16x8 커널로 실행하면 이미지당 9,641ms가 걸려, 동일 빌드 조건(O3+LTO)의 스칼라 C 구현(4,494ms)보다 약 2.1배 느리다. 원인은 세 가지 갭이다. (i) 표준 스택에는 LIF(Leaky Integrate-and-Fire) 뉴런 연산자가 없다. (ii) CMSIS-NN s16 DSP fast 경로는 im2col 열 길이가 512 미만일 때만 적용되며, 본 모델의 두 번째 합성곱은 열 길이가 $5\times5\times32=800$이므로 일반 C 커널로 폴백한다. (iii) 레이어 퓨전이 만들어 내는 int16 가중치를 처리할 커널이 없다(CMSIS-NN s16 커널은 int8 가중치 전용).

본 논문의 접근은 프레임워크를 교체하는 대신, 기존 스택을 유지한 채 컴파일러 퓨전 패스와 커스텀 연산자 1개만을 추가하는 것이다. snnTorch와 Brevitas로 양자화 인지 학습(QAT)된 SNN을 CHACHA 컴파일러가 정수 전용 코드로 lowering하고, Conv+AvgPool을 $6\times6$/스트라이드 2 커널 하나로 대수적으로 퓨전하며, LIF까지 결합한 \texttt{SNN\_CONV\_POOL\_LIF} 커스텀 연산자가 이를 실행한다. 그 결과 동일한 인코딩·LIF 조건에서 비퓨전 TFLM 경로 대비 4.0배 빠른 이미지당 2,407ms를 얻었고(스칼라 대비 종단 1.87배, 입력 인코딩 개선 포함), TFLM 런타임 오버헤드는 동일 커널 직접 호출 대비 약 0.5\%에 불과함을 분리 실측하였다. 본 논문의 기여는 다음과 같다.

\begin{itemize}
\item \textbf{CHACHA 컴파일러}: snnTorch+Brevitas QAT SNN을 정수 전용(int8 가중치, Q15 활성, Q15 LIF 파라미터) MCU 코드로 lowering하는 모델 컴파일러. Conv+AvgPool 대수적 퓨전 패스, FC 가중치의 CHW$\rightarrow$HWC 빌드타임 퍼뮤트, C 배열/TFLite flatbuffer 이중 백엔드를 제공한다(III장).
\item \textbf{\texttt{SNN\_CONV\_POOL\_LIF} 커스텀 연산자}: 퓨전 가중치($|w|\le508$)가 int8에 담기지 않는 문제를 SMLALD 듀얼 $16\times16$ MAC과 64비트 누산 커널로 해결하고, requantize 직후 2픽셀 타일에 QADD16/SSUB16/SEL 브랜치리스 LIF 에필로그를 결합하여 conv/pool 중간 텐서를 완전히 제거한다(IV장).
\item \textbf{표준 커널 한계의 실측 규명과 회복}: CMSIS-NN s16 fast 경로의 적용 조건(im2col 열 길이 $<512$) 때문에 표준 경로가 스칼라 대비 약 2.1배 느려지는 현상을 STM32F746 실측으로 규명하고, 제안 퓨전 연산자로 동일 조건 4.0배 가속(2,407ms/이미지)을 달성한다(V장).
\item \textbf{비트 정확 검증과 배포 envelope}: 퓨전 항등식·커널·전체 파이프라인의 호스트 비트 일치 검증과 LIF SIMD 1,887만(18.87M) 조합 전수 검증을 수행하고, 메모리 비용 모델과 실측 처리량으로 본 보드에서 배포 가능한 모델 범위를 정량 제시한다(V--VI장).
\end{itemize}

본 논문에서 실기기로 검증한 모델은 LeNet-5급\cite{lecun1998gradient} SNN 1종(CIFAR-10\cite{krizhevsky2009cifar}, 94,666 파라미터)이다. 컴파일러와 커널의 구조가 허용하는 확장 범위는 구조적으로 논증하되, 실질 제약이 보드의 Flash/SRAM 용량임을 VI장에서 비용 모델로 정량화한다.

본 논문의 구성은 다음과 같다. II장에서 배경과 관련 연구를 정리하고, III장에서 CHACHA 컴파일러를, IV장에서 Conv+AvgPool+LIF 퓨전 연산자와 TFLM 통합·펌웨어 구현을 기술한다. V장에서 실측 결과와 비트 정확 검증을, VI장에서 배포 가능 영역 분석을 제시하고, VII장에서 결론을 맺는다.
