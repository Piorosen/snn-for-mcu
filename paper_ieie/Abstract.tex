Spiking neural networks (SNNs) repeat every layer over tens of timesteps, so on tiny microcontrollers this temporal loop dominates inference latency. On an STM32F746 with 320KB SRAM and 1MB Flash, we show that the standard TinyML stack---TensorFlow Lite for Microcontrollers (TFLM) with CMSIS-NN---has three gaps for SNNs: no LIF operator, an int16 DSP fast path restricted to im2col column lengths below 512, and no support for the int16 weights produced by layer fusion. The standard int16x8 path thus takes 9,641ms per image, 2.1x slower than plain scalar code (4,494ms), as its second convolution falls back to a generic C kernel. We present the CHACHA compiler, which lowers snnTorch+Brevitas QAT models to integer-only code and algebraically fuses Conv+AvgPool into one 6x6 stride-2 kernel, and a fused Conv-AvgPool-LIF custom operator using SMLALD 64-bit accumulation with a branch-free QADD16/SSUB16/SEL LIF epilogue. The fused operator achieves 2,407ms per image: 4.0x faster than the unfused TFLM path under identical conditions and 1.87x faster than the scalar baseline end-to-end, with about 0.5\% TFLM overhead and 80.3KB arena usage. Host tests confirm bit-exact correctness, including exhaustive verification of 18.87 million LIF SIMD combinations, and a memory cost model quantifies the deployable model envelope.
